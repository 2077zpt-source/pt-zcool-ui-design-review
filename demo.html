UI 设计稿评审助手

从信息层级、组件一致性、交互状态和视觉细节角度审查 UI 设计稿。

查看 SKILL.md GitHub 仓库

这是做什么的

帮助 UI/UX 设计师在交付前发现页面目标、信息层级、布局对齐、组件一致性、状态设计、文案和可访问性问题，并输出按优先级排列的修改清单。

适合谁

UI 设计师、UX 设计师、产品设计师、设计团队负责人

UI评审UX设计走查界面优化交付检查

典型场景

当你准备交付 UI 设计稿，想检查信息层级、组件一致性、交互状态和视觉细节时使用。

输入什么

产品类型、页面目标、目标用户、设备尺寸、当前设计阶段、重点关注、已知限制、截图或页面说明

输出什么

总体判断、P0-P3 问题清单、逐区块建议、文案优化、交付前检查表

使用流程

  1. 1输入页面目标
  2. 2检查信息层级
  3. 3按优先级列问题
  4. 4生成交付前检查表

示例

输入“SaaS 数据看板”，输出 P0-P3 问题清单、逐区块建议和验收标准。
